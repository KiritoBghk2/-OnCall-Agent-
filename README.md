**智能OnCall Agent系统**

**项目介绍：&#x20;**&#x667A;能OnCall系统通过AI Agent解决团队真实痛点，整合 **知识库、对话、运维&#x20;**&#x4E09;大核心能力，实现 **问题自动应答&#x20;**&#x548C; **故障智能排查&#x20;**&#x7684;一体化服务。致力于降低团队OnCall的人力成本，提升团队效率。


**技术栈&#x20;**：SpringBoot、SpringAI-Alibaba、RAG、ReAct、Plan-Executor、Multi-Agent、MCP

**个人职责：**

* **负责 AI Agent 架构设计&#x20;**：基于 Eino 框架设计并实现了多个AI Agent，包括 **Knowledge Index Agent、Chat ReAct Agent 和 Plan-Execute-Replan Agent，&#x20;**&#x901A;过图编排实现模块化的 Agent 工作流。 **设计并实现了多工具集成方案&#x20;**，包括通过 MCP 协议集成日志查询工具、Prometheus 告警查询工具、MySQL 数据操作工具、联网查询等，使 AI Agent 能够灵活调用外部工具完成复杂任务。

* **负责 RAG 知识库系统设计&#x20;**：设计了完整的 **文档向量化存储和检索方案&#x20;**，支持内部文档的智能检索增强生成。使 AI 能够基于内部知识库提供准确的业务咨询和技术支持。在 RAG 系统中，需要处理文档分割大小和检索TopK参数的选择。分割过小会丢失上下文，过大会降低检索精度；TopK 过大会影响性能，过小可能遗漏关键信息。经过测试验证，确定了最优参数组合， **知识检索准确率达到 85%+**

* **负责对话功能开发：基于 ReAct 模式实现了对话Agent 。&#x20;**&#x5B9E;现业务咨询、告警自救、工单预处理等场景无缝切换，一次开发覆盖研发、运维、业务方多角色需求，支持多轮对话上下文记忆，并通过容错处理优化体验。同时基于 **SSE技术实现AI对话流式输出&#x20;**，解决大模型响应延迟问题，前端呈现实时对话效果。

* **负责AIOps功能开发&#x20;**：基于 **Plan-Execute-Replan 模式&#x20;**&#x5B9E;现了 **智能运维Agent&#x20;**。实现了 **根据告警信息 → 检索知识库 → 规划执行步骤 → 调用工具查询 → 分析结果 → 生成建议&#x20;**&#x7684;完整业务流程， **解决了传统运维需要人工排障的低效问题，将运维响应时间从小时级降低到分钟级。&#x20;**&#x80FD;够自动根据内部文档查询监控信息和日志信息，通过工具调用实现自动化监控告警查询和日志分析，并结合历史工单生成运维建议方案。
